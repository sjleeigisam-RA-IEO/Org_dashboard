# IOTA Seoul Data Package v0.9

이 폴더는 IOTA Seoul 개발관리 Workspace 화면과 연동하기 위한 데이터 패키지입니다.  
기준일은 2026-04-30이며, v0.9는 추가 요청자료를 반영하되 계획안·검토안·시나리오 값은 확정값처럼 보이지 않도록 분리한 버전입니다.

## 파일 구성

| 파일명 | 목적 | 보는 방법 |
|---|---|---|
| `IOTA_Seoul_Master_DB_v0.9.xlsx` | 원본 스키마를 유지한 마스터 DB입니다. 각 테이블의 PK/FK 구조와 원천 추출, 미충전 필드를 함께 확인할 수 있습니다. | 데이터 검수자나 PM이 전체 DB 구조와 원천 반영 상태를 볼 때 사용합니다. |
| `IOTA_Seoul_DB_for_UI_v0.9.xlsx` | UI/UX 제작자가 화면에 바로 쓰기 쉽도록 정리한 버전입니다. | 화면 제작자는 이 파일을 우선 참고합니다. 시트별 데이터가 대시보드 컴포넌트에 들어갈 기본 값입니다. |
| `IOTA_Seoul_IFPDP_HandOff_v0.9.xlsx` | 제작 담당자 전달용 요약 데이터입니다. `요약 → 확정값 → 계획_시나리오값 → 충돌값 → 미충전값 → 원천추출 → 파일목록` 순서로 구성했습니다. | 화면 구성이나 디자인 시 필요한 값만 빠르게 찾을 때 사용합니다. 가장 먼저 열어보는 것을 권장합니다. |
| `v0.9_build_report.md` | v0.9 생성 결과, 반영/보류/충돌 건수, 주요 충돌, 검증 결과를 정리한 작업 보고서입니다. | 데이터 품질과 남은 확인사항을 볼 때 사용합니다. |
| `IOTA_Seoul_Data_Package_v0.9.zip` | 위 파일과 이 안내문을 묶은 다운로드용 압축파일입니다. | 팀원에게 한 번에 전달할 때 사용합니다. |

## 우선 확인 순서

1. `IOTA_Seoul_IFPDP_HandOff_v0.9.xlsx`의 `요약` 시트를 먼저 봅니다.
2. 화면 제작에 필요한 확정값은 `확정값` 시트에서 확인합니다.
3. 아직 사업상 확정되지 않은 값은 `계획_시나리오값` 시트에서 확인합니다.
4. 서로 다른 문서에서 값이 충돌한 항목은 `충돌값` 시트에서 확인합니다.
5. 화면에 넣으면 안 되거나 추가 확인이 필요한 값은 `미충전값`, `원천추출`, `v0.9_build_report.md`를 함께 봅니다.

## 주의사항

- Rescue/F2/정상화계획안/PF 추진계획안의 값은 문서에 존재하더라도 계획안·검토안·시나리오로 분리했습니다.
- 선매입확약 1,600억원과 조건부 매입확약 800억원은 충돌 그룹으로 묶었고, 확정값으로 승격하지 않았습니다.
- 국토부 보고 화면/다운로드에는 증빙·오너 컬럼을 노출하지 않도록 정리했습니다.
- 민감한 내부 협상 전략, 금리·수수료·특약, 임차 조건, 내부 반대 의견 등은 외부 제출용 화면에서 제외하거나 마스킹해야 합니다.
