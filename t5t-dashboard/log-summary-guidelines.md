# T5T Log Summary Rule

## 목적

원문 로그를 그대로 분류하지 않고, `프로젝트 매칭`, `이해관계자 파악`, `업무 성격 분류`에 유리한 형태로 먼저 정제한 뒤 Python 규칙이 이를 우선 해석하도록 한다.

## Notion 권장 컬럼

- `원문 요약`
  - 사람이 읽는 기준의 요약 문장
- `분류 요약`
  - 분류 친화형 구조화 요약
- `분류 토큰`
  - 해시태그 또는 나열식 키워드

`sync_notion.py`는 Notion 속성을 그대로 JSON으로 옮기므로, 위 컬럼을 추가하면 대시보드 파이프라인에서 바로 활용할 수 있다.

## 분류 요약 작성 규칙

한 문장 또는 두 문장 이내로 작성하되 아래 요소를 가능한 한 포함한다.

- 프로젝트명, 자산명, 펀드 코드(`***호`) 등 매칭 단서
- 상대방 회사명, 기관명, 담당 조직 등 이해관계자 단서
- 업무 성격이 드러나는 표현
  - 예: `매입 검토`, `셀다운 협의`, `인허가 대응`, `임대차 협상`, `LP 보고`, `PF 조달 검토`

좋은 예시:

`성수 오피스 개발사업 관련, 미래에셋과 선순위 PF 조달 조건 협의 및 3호 펀드 투자구조 검토 진행.`

피해야 할 예시:

`관련 회의 후속 검토 진행.`

## 분류 토큰 작성 규칙

해시태그 또는 구분자 기반 나열식으로 입력한다.

- 예시 1: `#성수오피스 #미래에셋 #PF #선순위대출 #3호펀드 #금융구조`
- 예시 2: `성수오피스, 미래에셋, PF, 선순위대출, 3호펀드, 금융구조`

권장 원칙:

- 프로젝트/자산 1~2개
- 이해관계자 1~3개
- 업무 성격 1~3개
- 너무 일반적인 단어는 제외
  - 예: `검토`, `진행`, `회의`, `관련`

## Python 반영 원칙

`server.py`는 아래 순서로 해석한다.

1. `분류 요약`
2. `분류 토큰`
3. `비고`
4. `업무 로그명`
5. 프로젝트 relation에서 해석한 프로젝트명

또한 `분류 토큰`은 일반 텍스트보다 우선 신호로 취급한다. 따라서 해시태그를 잘 넣을수록 카테고리, 키워드, 이해관계자 추출 정확도가 올라간다.
