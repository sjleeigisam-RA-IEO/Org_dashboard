# 🚀 IGIS 자산 인텔리전스 데이터 파이프라인 가이드

이 문서는 CRM 엑셀 데이터를 분석하여 Supabase 클라우드 데이터베이스에 적재하고, 대시보드에서 시각화하기까지의 전체 프로세스를 설명합니다.

---

## 1. 데이터 업데이트 흐름 (Pipeline Overview)

전체 업데이트는 `run_update.bat` 실행 시 시작되며, 다음 4단계를 거칩니다.

1.  **파일 분석 (Analysis):** `processor.py`가 폴더 내 최신 엑셀 파일을 자동 식별하고 파싱합니다.
2.  **데이터 정제 및 지능화 (Refinement):** `geocoder.py`를 통해 주소를 정제하고 PNU(자산고유번호) 및 모펀드 ID를 추출합니다.
3.  **데이터베이스 적재 (Upload):** `uploader.py`가 중복을 방지(Upsert)하며 Supabase에 데이터를 전송합니다.
4.  **시각화 (Visualization):** `app.js`가 고유 코드를 기반으로 데이터를 그룹화하여 대시보드에 표시합니다.

---

## 2. 상세 로직 설명

### 📂 파일 분석 및 매핑 (`processor.py`)
*   **파일 자동 식별:** 정규표현식을 사용하여 `펀드 관리_*.xlsx`, `대주단_*`, `수익자_*` 등의 패턴을 가진 최신 파일을 찾습니다.
*   **컬럼 매핑:** `mapping.json` 설정을 기반으로 엑셀의 한글 컬럼명을 DB의 영문 필드명으로 변환합니다.
*   **지능형 추출:** 펀드 마스터 데이터 추출 시, 특정 인덱스(53번 열)를 추적하여 **모펀드(Parent Fund) 코드**를 확보합니다.

### 🏠 주소 정제 및 PNU 추출 (`geocoder.py`)
*   **VWorld API 연동:** 입력된 주소를 도로명/지번 기반으로 정밀 검색합니다.
*   **PNU 확보:** 검색 결과에서 해당 토지의 고유 식별자인 **PNU 코드**를 추출합니다. 이 코드는 서로 다른 명칭의 자산을 하나로 묶는 핵심 연결 고리가 됩니다.
*   **건축물대장 연동:** 국토교통부 API를 호출하여 연면적, 층수, 주차대수 등 상세 제원을 수집합니다.

### ☁️ DB 적재 및 중복 방지 (`uploader.py`)
*   **Upsert 전략:** `on_conflict` 옵션을 사용하여 이미 존재하는 데이터는 업데이트하고, 없는 데이터만 새로 생성합니다.
    *   *펀드:* `fund_id` 기준
    *   *자산:* `fund_id + asset_name` 기준
*   **메타데이터 임베딩:** DB 스키마를 변경하지 않고도 새로운 정보를 저장하기 위해, PNU와 모펀드ID 등을 `metadata` (JSONB) 필드에 통합하여 저장합니다.

---

## 3. 데이터베이스 스키마 명세 (Key Tables)

### 📊 `funds` (펀드 마스터)
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `fund_id` | VARCHAR (PK) | 펀드코드 (예: 112421) |
| `fund_name` | VARCHAR | 펀드 전체 명칭 |
| `short_name` | VARCHAR | 펀드 약칭 (예: 353호) |
| `metadata` | JSONB | **모펀드ID**, 기타 속성 저장 |

### 🏢 `fund_assets` (자산 정보)
| 필드명 | 타입 | 설명 |
| :--- | :--- | :--- |
| `fund_id` | VARCHAR | 연결된 펀드 코드 |
| `asset_name` | VARCHAR | 자산(건물) 명칭 |
| `address` | TEXT | 정제된 도로명/지번 주소 |
| `gfa` | NUMERIC | 연면적 (Gross Floor Area) |
| `metadata` | JSONB | **PNU 코드**, 건축물 상세 제원 저장 |

---

## 4. 대시보드 스마트 그룹화 (`app.js`)

데이터가 파편화되어 있어도 대시보드에서는 다음과 같은 우선순위로 데이터를 묶어서 보여줍니다.

*   **자산 그룹화:** `metadata.pnu` (PNU 코드) 가 같으면 물리적으로 같은 건물로 판단하여 하나의 카드로 통합.
*   **펀드 그룹화:** `metadata.parent_fund_id` (모펀드 코드) 가 같으면 하나의 펀드 시리즈로 판단하여 통합.

---

## 5. 유지보수 방법

새로운 엑셀 데이터가 입수되면 다음 과정을 수행하세요.

1.  **파일 복사:** 새로운 엑셀 파일을 작업 폴더에 넣습니다. (이전 파일은 지우지 않아도 무방합니다.)
2.  **배치 실행:** `run_update.bat`을 더블 클릭합니다.
3.  **확인:** 터미널 창에 "All Data synced!" 메시지가 뜨면 완료입니다.
