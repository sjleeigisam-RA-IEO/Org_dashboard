# ⚠️ 데이터 정합성 오류 분석 보고서 (2026-04-24)

## 1. 발견된 문제
- **현상**: '서울로타워' 검색 시 실제 관련 없는 '와이디816(P00037)' 펀드가 동일 자산 참여자로 그룹화되어 노출됨.
- **영향**: 대시보드상 자산별 투자자 현황 및 익스포저 집계 오류 발생 가능성.

## 2. 원인 분석 (Root Cause)
- **대상 파일**: `투자 자산 조회_20260424.xlsx`
- **오류 지점**: 엑셀 데이터 673번 행 (Row 673)
- **세부 내용**: 
    - 와이디816 PFV(`P00037`) 레코드에 **오투타워**가 아닌 **서울로타워**의 정보(명칭 및 주소)가 기입되어 있음.
    - 동일 펀드(`P00037`)가 672번 행(오투타워)과 673번 행(서울로타워)에 중복 존재하며, 각각 다른 자산으로 매핑됨.

| 행 번호 | 펀드코드 | 자산명 | 주소 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| 672 | P00037 | 오투타워 | 영등포구 여의도동 23-6 | 정상 데이터 |
| 673 | P00037 | **서울로타워** | **중구 남대문로5가 541** | **데이터 기입 오류** |

## 3. 시스템 처리 결과
- **그룹화 로직**: 시스템은 PNU(필지번호)를 기준으로 자산을 묶기 때문에, 673번 행의 잘못된 주소를 기반으로 생성된 PNU가 진짜 서울로타워와 일치하여 한 카드로 통합됨.
- **상세 페이지**: 한 펀드가 두 자산을 보유한 것으로 인식되어, 상세 정보 호출 시 우선순위에 따라 오투타워 정보가 상단에 노출되는 부작용 발생.

## 4. 조치 권고 사항
- **데이터 수정**: 원천 업무시스템 내 `P00037` 펀드와 자산 간의 매핑 관계 재확인 및 수정.
- **단기 대응**: 차기 데이터 업로드 시 673번 행과 같은 오기입 데이터 삭제 후 재업로드 필요.
- **시스템 보완**: 동일 펀드가 주소가 완전히 다른 지역의 자산을 다수 보유할 경우 경고를 띄우는 유효성 체크 로직 검토.
